# 后端程序逻辑

当前项目没有后端服务。

## 当前状态

- 游戏是纯前端应用。
- 数据配置直接随前端代码发布。
- 战役存档使用浏览器 `localStorage`。
- 部署目标是 GitHub Pages 子页面。

## 约束

- 不要假设存在服务器、数据库、登录系统或远程存档。
- 涉及存档、设置、统计等持久化需求时，默认先考虑浏览器本地存储。
- 如果未来引入后端，应先补本文档，明确 API、鉴权、数据迁移和离线兼容策略。

## 未来可能的后端职责

- 云存档同步。
- 排行榜或战役统计上传。
- 用户配置同步。
- 远程平衡参数下发。

在这些功能真正出现前，不要在前端代码中预留复杂的后端抽象。
