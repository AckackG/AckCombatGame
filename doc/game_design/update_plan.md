# 更新计划

本文档集中记录未来更新方向。各设计文档只保留当前约束和设计原则，避免计划散落在多个文件里。

## P0：武器与战斗更新

- 后坐力机制深化：连续开火时后坐力随时间或弹药消耗增加，停火后衰减。
- 统一即时命中武器和子弹底层命中结算，避免两套伤害路径长期分叉。
- 追踪导弹：低射速、高可靠命中，适合打高速单位。
- 电弧/闪电武器：短中距离弹射多个目标，适合清群。
- 光束炮：直线贯穿或持续照射，需要明确命中判定和性能边界。
- 地震炮：范围震荡或近距离冲击，需要明确友伤和控制强度。
- 医疗枪：攻击友军并恢复生命，需要明确避免治疗敌军。
- 冰冻/减速武器：降低敌军推进速度，属于控场而非纯伤害。
- 引力/冲击武器：改变单位或子弹运动，需要优先定义友伤和性能边界。

## P1：单位与战役内容

- 掩体：长条建筑单位，用于改变战线和阻挡推进。
- 装甲车：高射速、较高机动，适合压制轻甲单位。
- 坦克：高血量、慢速、高单发或范围伤害，适合突破和阵地战。
- 战役结束复盘：显示队伍成长、武器贡献和关键波次表现。
- 战役掉落或购买池：区分玩家单位、炮塔、敌军和怪物的可用武器范围。

## P2：世界逻辑与性能

- 更新为空间索引 2.0，支持圆形、线段和多边形等更丰富的检索形状。
- 探索将单位间的物理碰撞排斥逻辑从全局世界循环移动到单位自身更新中，降低世界循环职责。
- Debug 显示密度控制：大量单位时优先保留关键状态，避免遮挡战场。
- 性能复盘：记录关键波次的单位、子弹、碰撞和渲染峰值。

## P3：系统设置与平台能力

- 系统设置：地图大小、音量、Debug 显示密度等参数可持久化到本地。
- 云存档同步。
- 排行榜或战役统计上传。
- 用户配置同步。
- 远程平衡参数下发。

## 执行约束

- 新武器先在沙盒验证，再进入战役随机池。
- 涉及战役、单位成长、武器池或存档的改动必须补回归测试。
- 引入后端前必须先更新 `doc/program_logic/backend.md`，明确 API、鉴权、数据迁移和离线兼容策略。
