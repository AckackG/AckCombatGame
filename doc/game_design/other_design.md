# 其他设计

本文档记录暂时不适合放入武器、世界逻辑或战役设计的体验与内容约束。

## UI 体验

- 主菜单采用战术终端风格，包含战役、沙盒和武器数据库入口。
- HUD 应优先显示战斗必要信息：FPS、金钱、波次、下一波倒计时、敌军状态。
- Debug 信息应避免遮挡战场。大量单位时，Debug 文本密度比完整性更重要。
- 战斗统计面板用于复盘武器表现，重点展示命中率、伤害效率、平均 DPS 与实战 DPS。

## 沙盒体验

- 沙盒用于验证武器、单位、怪物和性能，不应被战役平衡限制过度束缚。
- 沙盒中的 Debug 单位可以暴露更多武器和怪物，但需要清楚标识怪物专用或实验内容。

## 音效与反馈

- 生成、死亡、升级、开火、爆炸等反馈应帮助玩家理解战场，而不是单纯增加噪音。
- 同类高频音效需要复用对象池，避免浏览器音频创建导致卡顿。

## 未来内容方向

- 玩家单位：掩体、装甲车、坦克等更明确的角色分工。
- 系统设置：地图大小、音量、Debug 显示密度等参数可持久化到本地。
- 统计复盘：战役结束后显示队伍成长、武器贡献和关键波次表现。
