# 世界逻辑设计

本文档记录世界状态、移动、索敌和碰撞相关的设计约束。具体单位定位放在 `unit_design.md`，未来计划集中记录在 `update_plan.md`。

## 世界状态

- 世界由单位、子弹和视觉提示组成。单位与子弹分离管理，避免不同生命周期混在同一数组里。
- 战斗单位包括玩家单位、炮塔、基地、敌军士兵、怪物和测试 Dummy。
- 所有可被战斗系统处理的实体都应维持明确的阵营颜色，阵营判断目前依赖 `color`。

## 索敌与移动

- 单位默认自动索敌，玩家可通过 RTS 指令临时覆盖 AI。
- 索敌时优先使用距离平方比较，只有显示或物理距离计算确实需要时才开方。
- 怪物需要目标粘性，避免玩家通过反复拉扯导致怪物频繁切换目标。
- 手动控制时间内，玩家单位不应被自动 AI 拉走。

## 碰撞与空间划分

- 世界使用四叉树筛选附近单位和子弹，避免全量 O(n^2) 检查。
- 四叉树只负责宽松筛选，实体命中由自身碰撞形状决定；现有形状包括圆形、固定矩形和旋转矩形。
- 高速子弹使用上一帧到当前帧的线段与目标形状相交判定，避免一帧穿过目标却未命中的问题。
- 范围伤害按爆心到实体碰撞形状边缘的距离计算，不再只按圆心和半径计算。
- 不计算子弹与子弹碰撞，只计算子弹与单位、单位与单位。
- 沙袋、掩体和车辆会阻挡单位移动，但当前不做复杂寻路，只通过碰撞推出避免重叠。
- 投射物是否被阻挡由目标实体决定；普通单位保持无友伤，沙袋只挡敌方投射物，掩体挡双方投射物。
